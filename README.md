# System-Memory-Issues

User reports system suddenly stops working and shows a "Preparing Automatic Repair" screen. 

<h2>Skills and Tools: </h2>

-Command Prompts\
-System Hardware\
-Hardware Repair\
-Event Viewer\
-Windows Memory Diagnostics\
-Diagnostic and verification skills\
-User communication and customer-service skills\
-Structured troubleshooting methodology

<h2>Troubleshooting: </h2>

<h3>Identify the Problem: </h3>

User starts the system and begins to use it normally. At some point, the system bluescreens, flashes a message that says "Preparing Automatic Repair", and then cycles through a loop of this.

<h3>Establish a Theory of Probable Cause: </h3>

a. User is able to log into their account, so the issue isn’t immediate. Possibly a service starting or driver conflict. 

<h3>Test the Theory to Determine the Cause: </h3>

a. Start the system and check the event view. Windows logs > Systems is showing a memory error. Ran a Windows Memory Diagnostic and confirmed hardware issue.

<h3>Establish a Plan of Action and Implement the Solution:</h3>

1. Check system requirements for replacement memory.
2. Advise user as to the cost of replacement/potential for upgrade while doing so.
3. Replace the system ram following proper hardware repair procedures.


<h3>Verify Full System Functionality and Preventative Measures: </h3>

1. Reboot system and run another Memory Diagnostics test.
2. Test results come back normal, ask the user to use the machine as normal to test if issue reoccurs. 

<h3>Document Findings, Actions, and Outcomes: </h3>

1. Documentation (yup)
2. After letting the user test normal operations, I am relatively confident it was only a faulty memory issue. User was informed of results of testing and given instructions to reach out again if anything seems wrong.
